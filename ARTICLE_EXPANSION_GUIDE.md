# Руководство по расширению статей

## Требования к качеству

- **Минимум 1500 слов** - критическое требование
- **Минимум 3 уровня заголовков** (##, ###, ####)
- **Минимум 10 параграфов**
- **Минимум 2 списка** (маркированные или нумерованные)

## Стратегия расширения

### 1. Анализ текущей структуры

Перед расширением статьи:
1. Определите текущее количество слов
2. Определите недостающее количество слов (1500 - текущее)
3. Проанализируйте существующие разделы
4. Определите, какие разделы можно расширить

### 2. Элементы для добавления

**Дополнительные подразделы (###)**:
- Добавьте подразделы в существующие разделы
- Каждый подраздел должен содержать 150-200 слов
- Используйте конкретные примеры и данные

**Примеры и кейсы**:
- Добавьте реальные примеры компаний, продуктов, или сценариев
- Используйте конкретные цифры и статистику
- Включите исторические контексты

**Списки**:
- Преобразуйте часть текста в списки (преимущества, риски, характеристики)
- Добавьте списки примеров или рекомендаций
- Используйте маркированные списки для перечислений
- Используйте нумерованные списки для последовательностей

**Практические рекомендации**:
- Добавьте разделы с практическими советами
- Включите чек-листы или руководства
- Предоставьте конкретные шаги для действий

**Исторические контексты**:
- Добавьте информацию о развитии темы
- Включите ключевые вехи и события
- Объясните эволюцию концепций

**Сравнительные разделы**:
- Сравните различные подходы или технологии
- Используйте таблицы (в виде списков) для сравнения
- Объясните преимущества и недостатки

**Заключительные разделы**:
- Добавьте раздел "Looking Forward" или "Future Outlook"
- Включите практические рекомендации для инвесторов
- Предоставьте итоговые выводы

### 3. Типичная структура для расширения

```
## [Существующий раздел]
[Существующий контент]

### [Новый подраздел]
[Новый контент - 150-200 слов]

### [Новый подраздел]
[Новый контент - 150-200 слов]

## [Новый раздел]
[Новый контент - 300-400 слов]

### [Подраздел]
[Контент - 150-200 слов]

### [Подраздел]
[Контент - 150-200 слов]
```

### 4. Процесс расширения

1. **Откройте статью** в соответствующем файле автора
2. **Найдите поле `content`** статьи по `slug`
3. **Определите разделы для расширения**:
   - Разделы с недостаточным количеством контента
   - Разделы, где можно добавить подразделы
   - Места для добавления новых разделов
4. **Добавьте контент**, сохраняя:
   - Стиль и тон автора
   - Структуру и форматирование
   - Качество и релевантность
5. **Проверьте метрики**:
   - Количество слов >= 1500
   - Количество уровней заголовков >= 3
   - Количество параграфов >= 10
   - Количество списков >= 2


## Чек-лист для проверки качества

После расширения статьи проверьте:

- [ ] Количество слов >= 1500
- [ ] Количество уровней заголовков >= 3
- [ ] Количество параграфов >= 10
- [ ] Количество списков >= 2
- [ ] Нет placeholder текста
- [ ] Сохранен стиль автора
- [ ] Контент релевантен теме
- [ ] Структура логична и последовательна
- [ ] Добавлены примеры и конкретные данные
- [ ] Заключение содержит выводы

## Рекомендации по стилю

- **Используйте конкретные примеры**: Вместо общих утверждений, используйте конкретные примеры компаний, продуктов, или сценариев
- **Добавляйте данные**: Включайте статистику, цифры, и метрики где это уместно
- **Структурируйте информацию**: Используйте списки, подзаголовки, и параграфы для улучшения читаемости
- **Сохраняйте тон автора**: Поддерживайте стиль и тон, характерные для автора статьи
- **Обеспечьте плавность**: Новый контент должен естественно вписываться в существующую структуру
